# Схема metrics-YYYY.jsonl

jsonl не держит комментарии — поэтому схема живёт отдельным файлом. Одна строка jsonl = один день. `metrics-YYYY.jsonl` — **единственный источник чисел** во всей системе. Тексты дня (`tracking/daily/`) числа не дублируют.

Версия схемы: `v: 1`.

---

## Три градации полей

**Стрик висит ТОЛЬКО на FLOOR.** Пропуск CORE или OPTIONAL поля стрик НЕ рвёт — это обогащение картины, не условие.

### FLOOR — обязательно для стрика (30 секунд)
| Поле | Тип | Что значит |
|------|-----|-----------|
| `day_score` | 1-10 | главный субъективный балл дня (не равно настроению — это «как прожил») |
| `check_done` | `full` \| `min` \| `none` | уровень закрытого чека; `full`/`min` → стрик +1 |
| `tasks_planned` | 0-3 | сколько ключевых задач задумал |
| `tasks_done` | 0-3 | сколько закрыл (для % эффективности, не для стрика) |
| `day_type` | `normal` \| `rest` \| `freeze` | обычный день / осознанный отдых (стрик +1) / пауза (стрик сохраняется) |

### CORE — желательно, пропуск ≠ разрыв стрика
| Поле | Тип | Что значит | Кто читает |
|------|-----|-----------|-----------|
| `energy` | 1-10 | уровень энергии за день | NUDGE-условие, RECOVERY-тренд, линза Тело |
| `sleep_h` | число | часов сна | линза Тело (RECOVERY: сон↓), корреляции в недельной свёртке |
| `workout` | 0 / 1 | была ли физнагрузка | корреляции workout↔energy в недельной свёртке |

> **Не спрашивать числом.** CORE-поля коуч НЕ выпытывает («сколько спал? оценка настроения?»). Он извлекает их из потока надиктовки, если прозвучало («не выспался» → `sleep_h≈5`, «сходил на тренировку» → `workout:1`). Не прозвучало — `null`, не добираем. Это обогащение картины, не анкета.

### OPTIONAL — по ситуации, сигналы для линз
| Поле | Тип | Что значит | Кто читает |
|------|-----|-----------|-----------|
| `anxiety` | 1-10 | тревога | ≥8 → CRISIS, 5-7 → сигнал RECOVERY, линза Эмоц. регуляция |
| `meaning` | 1-10 | ощущение смысла (НЕ заставлять оценивать каждый день — невротизирует) | ≤2 → CRISIS, ↓ → RECOVERY/линза Смысл |
| `focus_blocks` | int | число блоков глубокого фокуса | линза Фокус/продуктивность (↓) |

> **Каждое поле читается — иначе его нет.** Поле живёт в схеме, только если на него кто-то реагирует по-разному (принцип «данные ведут к действию», `SPEC §1`). Поля без читателя (`mood`, `meditation`) убраны 2026-05-23: коуч их писал, никто не читал — это был сбор чисел ради чисел.

**Служебное:** `v` (версия схемы, нет поля = `null`) · `date` (ISO).

**Логический день — `day_rollover: 05:00`.** Чек, закрытый до 05:00, относится к ПРЕДЫДУЩЕй календарной дате (тот, кто работает за полночь, закрывает «вчера», а не открывает «сегодня» пустым днём). Окно 00:00–05:00 → `date` предыдущего дня. Важно для корректности стрика: иначе ночной чек создаёт лишнюю дату и рвёт цепь.

**A+ задача дня — это важнейшая из трёх ключевых; отдельным полем не пишется, живёт как первая в тройке.** Покрыта `tasks_planned` / `tasks_done` — отдельный счётчик не нужен.

---

## Правила записи

- Пропущенное поле = `null` или его просто нет в строке. Никаких заглушек.
- Пропущенный день БЕЗ заморозки = нет строки → стрик рвётся.
- Заморозка пишется авто-строкой (см. ниже) — стрик сохраняется.
- **Любая эмо-цифра — повод прочитать текст дня, не вывод.** `meaning: 8` при тексте «не понимаю зачем» → верим тексту.

---

## Примеры строк

Полный день:
```json
{"v":1,"date":"2026-05-23","day_score":7,"check_done":"full","tasks_planned":3,"tasks_done":2,"day_type":"normal","energy":7,"sleep_h":7,"workout":1,"anxiety":4,"meaning":8,"focus_blocks":3}
```

Floor-день (закрыты только 5 обязательных):
```json
{"v":1,"date":"2026-05-24","day_score":5,"check_done":"min","tasks_planned":3,"tasks_done":1,"day_type":"normal"}
```

Заморозка (авто-строка, день не заполняется):
```json
{"v":1,"date":"2026-05-25","check_done":"none","day_type":"freeze"}
```

## Арифметика тренда (всё на лету из jsonl)

Ничего не кешируется, кроме сводки в `weekly/` (cache-блок — легитимен по решению v4, реестр строка 9: monthly читает кеши, не пересчитывает год; истина всё равно в jsonl) и кеша стрика в `commitments.md`. Год читается целиком дёшево.

- **Среднее за 7 дней:** сумма поля по существующим строкам последних 7 дат ÷ число этих строк (не ÷7 — делим на покрытие).
- **Дельта к прошлой неделе:** `среднее_текущей_7д − среднее_прошлой_7д`. Показываем со знаком и с пометкой покрытия, если строк меньше 7.
- **Стрик:** пересчёт по `check_done` / `day_type` сверху вниз по датам (логика — `commitments.md`). Кеш в commitments, истина — здесь.
- **% задач:** `Σ tasks_done ÷ Σ tasks_planned` за период (дни без плана в знаменатель не идут).
